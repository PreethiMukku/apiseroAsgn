Thin Client (Web Applications)
    Client -------- Server communication
    Client is typically Web Browser and requires network to interact with server
    Makes Http requests
    Url format:
      any protocol + host address + port number
      https://localhost:8080/index.html
    servers:
      Microsoft IIS (web server + c#/vb.net code on serverside)
       Java --> Tomcat(Web server + java code on server side)
       Web Logic
       JBoss
Client side and Server side validations
    Java script for client-side validations
    Jsp/Servelets for server-side validations (in java)
Markup languages
    Html - Hyper text markup language -> format and display of data
    Xml - Extensible markup language -> transport and store data


HTML:
    case in-sensitive
    Two sections -> head + body
    Elements :
      heading tags
      paragraph tag
      break tag
   Tables:
       table row
       table column
       table heading
   Forms:
      form tag
      form action -> specifies destination after successful submission of a form
      method -> http method (get/post)

Creation of dynamic web project:
   Create a dynamic web project
   Configure tomcat server
   create sample .html form file, .jsp file for validation
   Save and run the project
Xml:
   Extensible markup language
   Used for storing and transporting data
   supports Custom tags
JSON:
   Java script object notation
